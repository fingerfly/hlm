# 版本升级命令与中文术语计划

## 当前状态

- PlanClass: `historical-archive`
- OverallStatus: `completed`
- LastUpdated: `2026-03-16`

## 目标与约束

- 保持规则基线不变，不改动
  `[src/config/ruleBaseline.js](02product/01_coding/project/hlm/src/config/ruleBaseline.js)`。
- 版本升级命令统一为 `npm run deploy -- major|minor|patch|build`（沿用 `deploy` 命令名）。
- 术语统一范围仅限前端 UI，不改文档、测试标题、注释文案。

## 实施步骤

- 盘点并收敛版本升级入口：检查
  `[package.json](02product/01_coding/project/hlm/package.json)` 与
  `[scripts/deploy.js](02product/01_coding/project/hlm/scripts/deploy.js)`，
  确保参数提示、错误输出与命令示例匹配目标格式。
- 强化版本升级逻辑健壮性：在
  `[src/config/versioning.js](02product/01_coding/project/hlm/src/config/versioning.js)`
  保持现有模式（build 增 build 号、patch/minor/major 升语义版本并重置
  build），只补必要边界校验与一致性约束。
- 用 TDD 补齐测试：先在
  `[tests/unit/versioning.test.js](02product/01_coding/project/hlm/tests/unit/versioning.test.js)`
  与 deploy 脚本测试中添加失败用例，再实现最小改动使测试通过。
- UI 术语统一（简体中文）：仅修改
  `[public/index.html](02product/01_coding/project/hlm/public/index.html)` 与
  `[public/app.js](02product/01_coding/project/hlm/public/app.js)` 中用户可见
  文案，确保麻将术语标准、简体、前后一致。
- 全量验证与规模检查：执行全测试（unit/spike/regression/integration），并对改动文件运行 `cloc`，确保文件与函数规模符合你的 SLOC/复杂度约束。

## 验收标准

- `npm run deploy -- major|minor|patch|build` 均按预期更新版本与 build。
- 版本升级相关测试覆盖非法参数、build 模式、语义版本三种升级模式。
- 前端页面中所有用户可见术语为统一的标准简体中文。
- 全测试通过，且未改动规则基线文件。
